HLS Audio/Subtitle Root Causes
• Audio: HLS.js needs startLevel set and audio codec hints configured. Some streams also need a CORS proxy pass-through.
• Subtitles: HLS.js subtitle display requires renderTextTracksNatively: false + a custom cue renderer overlay. Without this, subtitles are detected but never shown visually.
• Continue Watching tiles: Currently renders blank boxes because the poster URL is not fetched. Fix: call fetchMeta() for each item (already done for names — extend to include poster).
